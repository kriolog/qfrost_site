#### Версия 0.19.0 от 06.09.2015 ####

  В этом выпуске добавлен тренд температуры, сглаживание температур по среднемесячным, проведены работы
  над интерфейсом, английской локализацией программы и производительностью.

  Формат файлов .qfrost изменён, но поддерживается загрузка файлов предыдущей версии.
  
  Новый функционал:
    - Добавлен новый стиль блоков: разница между температурой и температурой начала фазовых переходов.
    - Возможно использование сглаживания температур по среднемесячным (при помощи квадратичного сплайна).
    - Добавлена поддержка тренда потепления. Пока что он применяется ко всем месяцам.

  Прочее:
    - Проведены работы над интерфейсом построения графиков по срезам
    - Добавлено 3 недостающих иконки в теме для Windows
    - Улучшен интерфейс задания граничных условий (добавлен график, две таблицы сведены в одну)
    - Косметические работы над статусной панели: к ползунку масштаба добавлены иконки, улучшен её общий вид
    - Исправлены ошибки в английской локализации программы - спасибо Максиму Торгонскому
    - Теперь блоков может быть в 5 раз больше (до 100 000) без потери производительности - спасибо Максиму
    - Цветовая шкала теперь вмещает текст вне зависимости от шрифта
    - Двойной правый клик теперь может центровать по активному инструменту
    - Улучшены скрипты 2D-построений: талая зона выделяется штриховкой и пр.
    - Доработан интерфейс задания кривой незамёрзшей воды - теперь к ней возможно добавлять новые точки
    - Максимальная влажность увеличена до 9999.99%
    - Добавлен разряд к плотности теплового потока (теперь его возможно устанавливать с точностью до 0.001)

#### Версия 0.18.1 от 14.01.2015 ####

  В этом незапланированном релизе исправлен расчёт кол-ва тепла, выделяемого внутренними источниками за шаг.
  Остальные изменения или мелкие/косметические, или подготавливают базу для расширения функционала в будущем.

  Исправление ошибок:
    - Корректная формула кол-ва тепла, передаваемого блоку от внутренних источников на каждом шаге по времени

  Прочее:
    - Графическая подложка: если привязка даёт неравномерное масштабирование, требуем подтверждения установки
    - Новые скрипты (построитель и диалог на PyQt) для настраиваемой 2D визуализации по txt-файлам, созданным
      через реализованный в 0.18.0 экспорт для построения (скоро диалог сможет вызываться прямо из программы)
    - Python-модуль для интерполяции периодической функции, заданной средними значениями; реализованный метод
      позволит менять параметры граничных условий пошагово или посуточно - а не резкими ежемесячными скачками
      (+ скрипт, демонстрирующий "ежегодный" интерполяционный сплайн, заданный среднемесячными температурами)
    - Новая иконка для полигонального выделения - символическое изображёние линзы грунта вместо пятиугольника
    - Иконки инструментов лучше сглажены: скрипт генерации PNG отныне не использует уменьшение растра 512x512
    - Исправлены практически все участки кода, которые вызывали дополнительные предупреждения компилятора gcc

#### Версия 0.18.0 от 23.12.2014 ####

  Прошёл год и день после выпуска 0.17.9, предыдущей версии QFrost. Приближается выход значимой версии - 1.0.
  Она будет полностью пригодна для инженерных изысканий как функционально (полноценная визуализация и проч.),
  так и юридически (к выходу 1.0 будут получены сертификат соответствия РСН 67-87 и авторское свидетельство).

  Направление развития при подготовке релиза указали специалисты из ОАО Фундаментпроект, использующие QFrost.
  Выпуск приурочен к началу обучения автора в аспирантуре кафедры геокриологии геологического факультета МГУ.

  Это первый выпуск после начала использования в процессе разработки программы системы контроля версий (Git).
  Теперь для всех новых выпусков QFrost будет составляться подробный список изменений вроде нижеприведённого.

  Новый функционал:
    - Добавлены новые способы применения грунтов: докраска (применение к пустым блокам в выделении) и заливка
    - Графическая подложка: возможность загрузки изображения (с его координатной привязкой) для выделения ИГЭ
    - Возможность построения графиков по выбранному срезу в расчётной области - с экспортом .png, .pdf и .txt
    - Инструмент эллиптического выделения блоков, необходимый в некоторых задачах (по просьбе С.Н. Булдовича)

  Исправление ошибок:
    - При экспорте результатов журналирования в CSV сохраняются обе координаты центров (а не дублирующаяся Y)
    - Диалоги теперь не дописывают тире и название программы к заголовку: Qt это давно проводит автоматически

  Прочее:
    - Файловые диалоги: при их запуске теперь открывается папка, задействованная в последнем принятом диалоге
    - Статусная панель: ползунок масштаба; рамки для виджетов с индикаторами, координатами курсора и привязки
    - Панель инструментов: кнопки выбора инструментов разбиты по группам, их настройки убраны в отдельный док
    - Полный экспорт расчётной области в текстовый файл и скрипт на Python, строящий 2D при помощи matplotlib
      (следующим этапом будет удобная для обычного пользователя кнопка "Построить поле"; ждите неделю-другую)
    - Привязка инструментов (anchor) теперь определяет созданные при помощи эллипса границы расчётной области 
    - Макс. мощность внутренних источников теплопотока увеличена на два порядка (не хватало даже для бетона!)
    - Проведена чистка и реорганизация кода - отделено 3 модуля, трансформации View снесены во ViewBase и пр.
